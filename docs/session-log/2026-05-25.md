## Decisions made

- **GIF only, not MP4 / webm** — universal sharing (iMessage, Discord)
  beats format quality for a 12yo's audience. FFmpeg.wasm rejected
  (30 MB bundle hit).
- **Single persisted animation, not many** — Emmett can export GIFs to
  archive; reduces cognitive load. Multi-animation gallery deferred.
- **Frame auto-creation on ▶▶ at end of timeline** — no separate "Add
  Frame" button. More intuitive for a kid.
- **History resets when switching frames** — simpler than per-frame
  undo stacks; documented as a known trade-off (see FORWARD_PLAN).
- **Render Static Site over GitHub Pages / Cloudflare Pages** — user
  already has Render account + render.yaml is canonical for repeatable
  deploys.
- **Transparent canvas + separate white bg layer** — required for
  onion skin to render correctly underneath; affected `toBlob` API
  split into `toFrameBlob` (transparent) vs `toExportBlob`
  (composited).
- **gifenc over gif.js** — modern, ES module, smaller, no web worker
  setup friction with Vite. Trade-off: less common in tutorials.
- **Personalization is a feature, not noise** — welcome modal
  intentionally calls out Emmett by name; this is his app, not a
  generic tool. Memory saved at
  [memory/project_flameanimations.md](../../../.claude/projects/-Users-thatcardboardlaptop-Projects-MANUS-Document-Repository/memory/project_flameanimations.md).
